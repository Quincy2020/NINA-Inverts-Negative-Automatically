一、 v1.0 核心功能清单（MVP 架构）1. 输入与图像引擎ARW/RAW 线性读取： 集成 LibRaw (rawpy)，严格关闭所有 Gamma 2.2、相机白平衡和色彩矩阵，只读出最原始的 16-bit 线性 RGB 数据。双层画布机制（性能优化）： * 计算画布： 内存中保留一个 float32 的全尺寸线性矩阵，用于最终的高精度导出。预览画布： 缩放到 2K 分辨率的矩阵，用于 UI 界面的实时滑块响应。

2. 核心算法层（去色罩与反转）基础去色罩： 用户指定（或自动检测）片基颜色 $C_{mask}$，执行线性空间除法 $I / C_{mask}$。物理反转： 执行 $1.0 - I$ 得到正片。分通道直方图自动拉伸（AWB 兜底）： 反转后，R、G、B 通道独立裁剪掉 0.1% 的极端高光/暗部，并拉伸至 $[0, 1]$ 全量程。

3. 基础基础调色板（参数化控制）基本滑块： 曝光度（Linear 乘法）、对比度（中点拉伸）。黑白点切分： 黑点/白点滑块（或吸管工具）。吸管手动灰点： 提供一个吸管，用户点击画面里的中性灰（如路面、灰墙）自动校正白平衡（覆盖自动算法）。5. 输出与导出16-bit TIFF 导出： 将最终的计算画布应用 sRGB 色彩矩阵和标准的 Gamma 2.2 曲线后，导出为高品质的 TIFF 或 JPEG。

4. 智能辅助层（CV 检测）自动框检测： 利用“中值滤波 + 自适应二值化 + 4顶点轮廓逼近”自动定位成像区域。自动透视校正： 拿到 4 个角点后，一键将画面拉直（纠正翻拍时的微小倾斜）。顺藤摸瓜采片基： 沿框边缘外扩 $N$ 像素，自动提取一小块区域的中位数作为 $C_{mask}$。

=============== 
去色罩算法部分
=============== 

经典稳定的去色罩与反转算法步骤目前成熟的工业软件（如 SilverFast 或 Lightroom 插件 NLP）底层均采用类似的算法通路：

步骤一：采样未曝光片基（获取色罩基准）算法首先需要在线性空间下获取纯橙色片基（例如齿孔间隙或片头）的 RGB 向量。令这个基准向量为：$$C_{mask} = [R_{mask}, G_{mask}, B_{mask}]$$

步骤二：线性空间下的色罩消除（归一化）数码相机获取的原始数据是线性的 RAW 信号 $I_{raw} = [R, G, B]$。由于色罩在线性空间中与图像信号是相乘的关系，算法通过除以片基颜色来将色罩消去：$$I_{neutral} = \left[ \frac{R}{R_{mask}}, \frac{G}{G_{mask}}, \frac{B}{B_{mask}} \right]$$经过这一步，未曝光的片基部分在 $I_{neutral}$ 中变为了纯白色 $[1.0, 1.0, 1.0]$，同时完成了初步的通道白平衡。

步骤三：反转为正片（物理反转）完成色罩剥离后，进行数学反转。因为在线性空间下，最大值（片基）代表无曝光，在正片中应该对应纯黑。因此反转公式为：$$I_{positive} = 1.0 - I_{neutral}$$此时，原本的橙色片基变成了纯黑色 $[0.0, 0.0, 0.0]$，画面正式进入正片色彩空间。

步骤四：对数密度映射与通道独立拉伸（核心稳健点）胶片的感光特性符合 H&D 曲线（Sensitometric Curve）。在物理上，负片记录的是对数密度（Density）：$$\text{Density} = \log_{10}\left(\frac{1}{\text{Transmittance}}\right)$$如果直接对 $I_{positive}$ 应用标准的显示器 Gamma 2.2，画面会非常灰暗，色彩也会断层。极其稳健的算法会执行以下操作：转换到对数空间（模拟密度）： 将图像转入 Log 空间，在此空间下进行直方图操作。分通道直方图自动拉伸（Channel-independent Stretching）：由于不同胶片（如 Kodak Gold 或 Fuji C200）的三层乳剂 $\gamma$ 斜率不同，加上冲洗时的化学偏差，算法需要独立计算 R、G、B 三个通道的直方图，裁剪掉两端极少数的噪点像素（如丢弃前 0.1% 的暗部和后 0.2% 的高光），然后将各通道独立拉伸至 $[0, 1]$ 全量程。这一步能自动修正 90% 以上的偏色。
